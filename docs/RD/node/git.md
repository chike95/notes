# Git

常用的标识如下：

类型	描述

feat	新功能（feature）

fix	修补bug

docs	文档（documentation）

style	格式修改（不影响代码运行的变动）

refactor	重构（即不是新增功能，也不是修改bug的代码变动）

test	增加测试

chore	构建过程或辅助工具的变动，非src和test的修改，比如构建流程, 依赖管理等

perf	性能优化(performance)

improvement	改进

build	打包

ci	持续集成

revert	撤销，版本回退
